## 中文
- 修复 macOS 桌面发布仅构建 `dmg` 导致 updater 产物不会持久生成的问题，改为在 GitHub Release 工作流中同时构建 `app,dmg`，确保 `.app.tar.gz` 与签名文件可被后续发布步骤稳定收集。
- 保留上一版引入的跨平台 Node 资产整理逻辑，继续统一处理 macOS、Windows、Linux 三平台的桌面安装包、updater 产物与签名文件，减少 workflow 中的 shell 特化分支。

## English
- Fixed the macOS desktop release pipeline where building only `dmg` did not persist updater artifacts by switching the GitHub Release workflow to build `app,dmg`, ensuring `.app.tar.gz` and its signature are available for downstream release steps.
- Kept the cross-platform Node-based asset preparation introduced in the previous release so macOS, Windows, and Linux installer bundles, updater artifacts, and signatures continue to be normalized without fragile shell-specific workflow branches.
