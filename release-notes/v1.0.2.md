## 中文
- 修复发布脚本在 Linux 等非 macOS 环境下二次打包插件 zip 时可能携带过深目录路径的问题，确保解压后可以直接得到可导入浏览器的扩展目录。
- 调整插件发布产物生成逻辑，支持在 GitHub Actions 临时目录内直接构建并上传 release zip，同时保留本地 `release/v<version>` 预检路径，避免依赖 Plasmo 原始 zip 导致交付形态不稳定。
- 修复 GitHub Release 工作流的桌面端产物整理与上传流程，确保 macOS、Windows 和 Linux 的桌面客户端能进入正式 release。
- 将仓库脚本、Docker 构建和 GitHub Actions 使用的 Node.js 版本统一到 `24.14.0`，消除本地与 CI 版本漂移。
- 优化移动端工具切换交互，并补齐共享 `ToolSelect` 导出与嵌入式样式支持。

## English
- Fixed the release packaging script so extension zips created on Linux and other non-macOS environments no longer capture overly deep directory paths, allowing the extracted folder to be imported into the browser directly.
- Updated extension release packaging so GitHub Actions can build and upload the release zip directly from a temporary workspace path while keeping the local `release/v<version>` flow for manual inspection, avoiding any dependency on the raw Plasmo zip shape.
- Fixed the GitHub Release workflow for collecting and uploading desktop artifacts so the macOS, Windows, and Linux desktop clients are included in the final release.
- Unified the Node.js version used by repository scripts, Docker builds, and GitHub Actions to `24.14.0`, removing local-versus-CI drift.
- Improved the mobile tool-switching experience and completed the shared `ToolSelect` export plus embedded styling support.
