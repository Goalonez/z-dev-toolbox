## 中文
- 修复 Web Docker 发布在 `linux/arm64` 路径下使用 QEMU 执行 Node 构建时可能触发 `Illegal instruction` 并长时间卡住的问题，改为让前端构建阶段固定运行在 BuildKit 的原生 builder 平台。
- 修复 macOS 桌面 release 资产整理时把 updater 签名文件错误命名为 `.dmg.sig` 的问题，确保 `latest.json` 能正确识别 `macos/aarch64` 的 `.app.tar.gz` updater 产物与签名文件。

## English
- Fixed the Web Docker publish pipeline hanging on `linux/arm64` when QEMU executed the Node build and could fail with `Illegal instruction` by forcing the frontend build stage to run on the native BuildKit builder platform.
- Fixed the macOS desktop release asset normalization bug that renamed the updater signature to `.dmg.sig`, so `latest.json` now picks up the `macos/aarch64` `.app.tar.gz` updater artifact and signature correctly.
